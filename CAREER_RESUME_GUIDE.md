## 1. 项目一句话版本（简历抬头可用）

- **全栈版**：独立实现了一个 AI 驱动的成长管理系统（Vue 3 + TypeScript + Pinia + Node 预览后端 + Spring Boot 生产后端），覆盖任务管理、情绪重构、聊天辅导、成长结算与周报生成。
- **前端版**：负责搭建 Vue 3 + TypeScript 前端架构（Pinia + Router + i18n + PWA），实现看板/聊天/报告等核心页面与统一 API/状态管理。
- **Java 后端版**：实现 Spring Boot + JWT + JPA 的后端分层架构，提供认证、看板、结算、日志与偏好接口，并完成 Node/Spring 双后端契约兼容。

## 2. 项目描述模板（可直接放简历）

### 模板 A（社招中级偏结果）

**Seichou-Logos（个人全栈项目）**  
技术栈：Vue 3、TypeScript、Pinia、Vite、Node.js、Spring Boot、PostgreSQL、JWT

- 设计并实现双后端模式（Node 预览 + Spring Boot 生产），统一前端 `/api` 调用，提升本地开发与联调效率。
- 搭建 store-centric 前端数据层（`authStore`/`boardStore`/`logStore`/`preferenceStore`），将页面逻辑与数据副作用解耦，增强可维护性。
- 落地 AI 相关业务闭环：情绪重构、聊天辅导、任务结算、周报生成，并实现无 Key 场景 fallback，保证演示稳定性。
- 构建删除链路可靠性方案（路由健康检查、本地隐藏兜底、重试永久删除），降低后端不一致导致的用户感知失败。
- 实现 Node 预览后端本地持久化（文件落盘）与热更新干扰治理（watch ignore），解决“操作后自动刷新/数据回弹”问题。

### 模板 B（校招/初级偏能力）

- 从 0 到 1 搭建 Vue 3 + TypeScript 项目结构，完成路由守卫、全局状态、国际化和 PWA 注册。
- 使用统一 `apiFetch` 封装鉴权与错误提示机制，减少重复请求代码并统一异常体验。
- 在 Spring Boot 中实现 `controller -> service -> repository -> entity` 分层，完成 JWT 鉴权和核心业务接口。
- 针对删除失败、刷新回弹等真实问题做工程化修复：接口探测、前端兜底、重试同步、持久化落盘。

## 4. 面试讲解 90 秒脚本（可背）

“这个项目我主要解决了两件事：第一是**业务闭环**，第二是**工程可靠性**。  
业务上，我做了任务看板、情绪重构、聊天辅导、周报生成，并保证 Node 和 Spring 两套后端都能兼容前端契约。  
工程上，我把前端做成 store-centric 架构，用统一 `apiFetch` 管理鉴权和错误处理。后面在真实联调里遇到删除失败和刷新回弹问题，我做了删除链路健康检查、本地隐藏兜底、重试永久删除，以及 Node 持久化与 Vite 监听隔离。  
结果是：项目从“能演示”升级到“可持续调试和可恢复”，这也是我在工程化方面最有价值的成长。”
